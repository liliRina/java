В этом проекте используются:
Hibernate, Hikari и JPA — для работы с базами данных;  
JUnit5 — для тестов;  
Lombok — для сокращения шаблонного кода;  
Validation — для проверки входных данных;    
Jackson — для загрузки данных из .ndjson файла;  
Spring (Boot, MVC, Data JPA) — для запуска проекта, работы с контроллерами, репозиториями и спецификациями.

Цель приложения: реализация поиска косметических продуктов по комбинации ингредиентов в загружаемом пользователем каталоге.

Архитектура: классический MVC с разделением на слои (Entity, Repository, Service, Controller).

Веб-интерфейс:
1) /. Задание пути к ресурсу (.ndjson) и загрузка данных с переходом на /сhoosing. В процессе загрузки данных осуществляется валидация. Строки, не прошедшие валидацию, пропускаются.  
2) /choosing. Создание списка компонентов для поиска. Действия на странице: добавить компонент, очистить список, начать поиск.  
3) /found_products. Отображение результатов поиска комбинации ингредиентов в косметических продуктах.

Тестирование:
1) Тест для проверки поиска по комбинации компонентов.  
2) Параметризованный тест для проверки валидации.
